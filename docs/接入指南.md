# agent-memory 接入指南

| 项 | 内容 |
|----|------|
| 适用版本 | CLI **1.0.1** · schema **1.0.0** |
| 读者 | 要在 Cursor / Codex / Claude Code / Grok 等 Agent 中启用外挂记忆的人 |
| 相关 | [README](../README.md) · [使用手册](使用手册.md) · 根目录 `PROTOCOL.md` |

本文说明：**怎样把记忆系统接到具体 Agent、粘贴哪些规则、项目如何标记、沙箱如何放行、如何验收接入成功**。全文中文。

---

## 1. 接入在解决什么问题

| 没有接入时 | 接入之后 |
|------------|----------|
| 每个工具各记各的 / 或完全不记 | 多工具读写**同一记忆根** |
| 换窗口任务断掉 | `handoff` + 新会话 `context` 接续 |
| 偏好靠嘴说 | `remember` / T0 沉淀后自动注入 |
| 项目约定串味 | `project-detect` + scope 隔离 |

**重要：** CLI 只保证「读写文件规则正确」（L-Core）。  
Agent **会不会**每轮 `context`、会不会 checkpoint，取决于规则是否写进该工具的系统提示 / `AGENTS.md`（L-Protocol），模型仍可能偶尔漏调。

## 2. 接入前准备

### 2.1 安装 CLI

见 [使用手册 §1](使用手册.md)。确认：

```bash
agent-memory --version
# agent-memory 1.0.1 (schema 1.0.0)
```

### 2.2 准备记忆根

```bash
export AGENT_MEMORY_ROOT="$HOME/.agent-memory"   # 或你的自定义路径
agent-memory init
agent-memory doctor
```

**建议：** 记忆根放在**稳定路径**，不要放进会频繁删掉的临时目录。  
代码仓库（本 GitHub 项目）与**数据根**分离——数据不要提交到 git。

### 2.3 让 Agent 找得到命令

任选其一：

1. 在登录 shell 中配置 `PATH` 与 `AGENT_MEMORY_ROOT`（`~/.zshrc` 等）  
2. 在 Agent 规则里写**绝对路径**调用：
   ```bash
   /绝对路径/agent_memory/.venv/bin/agent-memory --root "/绝对路径/记忆根" <子命令>
   ```
3. 部分 GUI Agent **不继承** `~/.zshrc`：需在应用设置里配置环境变量，或一律用绝对路径。

### 2.4 `--root` 写法（避免接入翻车）

全局选项，**子命令前或后均可**（≥1.0.1）：

```bash
# 推荐
agent-memory --root "$AGENT_MEMORY_ROOT" context --query "关键词"

# 也合法（Agent 常这样拼）
agent-memory context --query "关键词" --root "$AGENT_MEMORY_ROOT"
```

## 3. 符合性等级（接入时必懂）

| 等级 | 你要做什么 | 失败算谁的 |
|------|------------|------------|
| **L-Core** | 安装正确、路径正确、CLI 绿测 | 实现 / 配置问题 |
| **L-Protocol** | 把下方「强制规则」粘进 Agent 常驻指令 | 模型漏执行 ≠ CLI 坏了 |
| **L-Ref** | 按本文粘贴 + 跑通演示 | 文档或粘贴不完整 |

## 4. 通用强制规则（粘贴块）

将以下 Markdown **原样或略改路径后**，粘贴到目标工具的常驻规则中  
（如 `AGENTS.md`、Cursor Rules、Claude 项目说明、Codex 说明等）。

```markdown
## 外挂记忆（agent-memory）

你与本机其它 Agent 共享同一套纯文件记忆库。

### 路径（按本机修改）

- 记忆数据根：`$AGENT_MEMORY_ROOT`（示例：`/Users/你/.agent-memory`）
- CLI：`agent-memory`（或写绝对路径：`…/agent_memory/.venv/bin/agent-memory`）
- 完整契约：记忆根内 `PROTOCOL.md`，或源码仓 `PROTOCOL.md`

```bash
export AGENT_MEMORY_ROOT="/你的/记忆根"
export PATH="/你的/agent_memory/.venv/bin:$PATH"
# 无 env 时：
# agent-memory --root "$AGENT_MEMORY_ROOT" <子命令>
```

### 强制节奏

1. **会话开始立刻**：  
   `agent-memory context --query "<用户消息关键词>"`  
   阅读输出中的 T0、Working、Semantic。  
   **Semantic 无命中时，禁止编造「记忆里有…」。**
2. **约每 8 条用户消息**，或重大决策 / 测试通过 / 准备换工具：  
   `agent-memory checkpoint --goal "…" --next-steps "- …"`  
   若已知项目：加上 `--project-id <id>`。
3. **用户要求记住 / 以后都 / 别再…**：  
   `agent-memory remember --slot <稳定槽位> --content "…"`  
   仅当前项目约定时加 `--project <id>`。
4. **用户要求忘掉**：  
   `agent-memory recent` 或 `search` 找到 id 后  
   `agent-memory forget <id>`。
5. **收工或换 Agent / 换窗口**：  
   `agent-memory session-end --title "…" --body "意图/动作/结果/教训"`  
   `agent-memory handoff --goal "…" --next-steps "- …"`  
   （可选）`extract --from <episode_id> --mode rules` 后审查候选。
6. **单活跃 working**：切换任务前先 handoff / session-end，避免互盖。

### 禁止

- 把 API 密钥、Token、Cookie、私钥写入记忆
- 把完整聊天记录当作 semantic
- 把模型猜测标成用户明确偏好（不要用 remember 记猜测）
- 在无 context/search 命中时声称「根据你的记忆…」

### 项目隔离

- 写项目语义前确认 `project-detect` 为 high，或使用仓库根 `.agent-memory-project`
- 不要把项目 A 的约定用到项目 B
```

按需把示例路径改成你的绝对路径。

## 5. 业务仓库侧最小接入

在**业务项目**（例如音乐 App 仓库）中增加：

### 5.1 项目标记文件

路径：仓库根 `.agent-memory-project`  
内容（一行）：

```text
my-app-id
```

仅含项目 id，无多余空格为佳。

### 5.2 可选说明文档

例如 `docs/agents/agent-memory.md`：写清本机 CLI 路径、记忆根、项目 id、常用命令。  
并在根 `AGENTS.md` 增加「必须先 context」的摘要（可引用上一节粘贴块）。

### 5.3 不要做的事

- **不要**把记忆数据目录放进业务仓库并提交  
- **不要**在业务仓库里再实现一套 memory  
- 记忆只存跨会话有用的约定与状态；**不能替代** PRD / ADR / 源码事实

## 6. 分工具说明

### 6.1 Codex（含 Desktop）

| 项 | 建议 |
|----|------|
| 规则位置 | 项目 `AGENTS.md`（建议走业务仓**版本更新**合入，勿临时强改）+ 可选说明文档 |
| **Hooks（推荐随记忆系统安装）** | 见下方 **§6.1.1 一键安装** |
| 环境变量 | Desktop 可能**不读** `~/.zshrc`；安装器会把 CLI/记忆根路径写入 `~/.codex/hooks/agent-memory/*.path` |
| 沙箱 | 记忆根在仓库外时，Agent **手动**写命令可能需升权；**hooks 在本机 shell 跑**，一般可直接写记忆根 |
| 验证 | 新会话后应有 `~/.codex/memory-cache/last-context.md`；回合结束后 `agent-memory recent` 有 checkpoint |

#### 6.1.1 Codex Hooks 一键安装（交付物）

仓库已写好 hooks 脚本与安装器，**使用者安装记忆系统后执行一条命令即可**（无需手改 `hooks.json` 拼路径）：

```bash
git clone https://github.com/valiantyan/agent_memory.git
cd agent_memory
python3 -m venv .venv && source .venv/bin/activate
pip install -e .

export AGENT_MEMORY_ROOT="${AGENT_MEMORY_ROOT:-$HOME/.agent-memory}"
agent-memory init   # 若尚未 init
bash scripts/install_codex_hooks.sh
```

| 安装结果 | 行为 |
|----------|------|
| `SessionStart` | `context` → **stdout 注入** Codex 会话；并写 `~/.codex/memory-cache/`（权限 700/600） |
| `Stop` | 消费 **`$AGENT_MEMORY_ROOT/meta/pending-turn/`**（由 `agent-memory turn` 写入）；完整才 checkpoint；否则 **no-op** |
| 已有 hooks | **命令级合并保留**（例如 Muxy 通知） |
| 卸载 | `bash scripts/uninstall_codex_hooks.sh`；可选 `--purge-cache` |

可选条件检索（用户消息含「上次/之前/怎么做」等）：

```bash
bash scripts/install_codex_hooks.sh --with-prompt-search
```

详情：[`scripts/codex-hooks/README.md`](../scripts/codex-hooks/README.md)。

**Codex 写入固定记忆示例话术：**

```text
记忆根在仓库外。请对下列命令使用可写权限（升权），执行全局固定记忆：

agent-memory --root "/你的/记忆根" remember --slot coding --content "只改任务相关文件；禁止顺手重构"
agent-memory --root "/你的/记忆根" remember --slot commit --content "中文 commit：原因、方案、影响范围"
agent-memory --root "/你的/记忆根" doctor
agent-memory --root "/你的/记忆根" context --query "coding commit"
```

### 6.2 Cursor

| 项 | 建议 |
|----|------|
| 规则位置 | User Rules / Project Rules / `.cursorrules` / `AGENTS.md` |
| 终端 | 集成终端若是 login shell，可继承 `~/.zshrc`；否则同样写绝对路径 |
| 验证 | Composer/Agent 新对话：是否自动 `context` |

### 6.3 Claude Code

| 项 | 建议 |
|----|------|
| 规则位置 | 项目 `CLAUDE.md` 或个人指令；可链到 `PROTOCOL.md` |
| 验证 | 新会话首轮工具调用是否包含 `agent-memory context` |

### 6.4 Grok / 其它可跑 shell 的 Agent

| 项 | 建议 |
|----|------|
| 规则位置 | 系统提示、Skill、或项目 `AGENTS.md` |
| 最低要求 | 能执行 shell + 读写记忆根（写操作需权限） |

### 6.5 不能跑 shell 的工具

v1 **不提供** MCP。只能：

- 人工在终端维护记忆，或  
- 换用支持 shell 的 Agent，或  
- 自行做薄封装（非本仓库范围）

## 7. 推荐会话协议（给人类看的时序）

```text
[会话开始]
  Agent: context --query "…"
  Agent: 根据 T0 / Working / Semantic 行动（无命中不编造）

[进行中]
  每 ~8 轮用户消息 → checkpoint
  用户说记住 → remember
  用户说忘掉 → forget
  重大决策 / 测通 → checkpoint

[换工具或收工]
  session-end（情节摘要）
  可选 extract → 人工 promote/reject
  handoff（给下一个 Agent）
```

**L-Core 不解析聊天流**，轮次计数与里程碑识别由 Agent/规则负责。

## 8. 沙箱与权限（接入高频坑）

| 操作 | 是否写盘 | 工作区外记忆根 + 只读沙箱 |
|------|----------|---------------------------|
| `context` `search` `get` `recent` `doctor` `project-detect` | 否（recent 自 1.0.1 只读） | 通常 **OK** |
| `checkpoint` `remember` `handoff` `session-end` `forget` `extract` `promote` `gc` `reindex` `init` | 是 | 常 **失败** → 需升权 |

**实践建议：**

1. 个人本机：在 shell 配置里 export 记忆根 + PATH，减少每条命令手写 `--root`。  
2. 严格沙箱工具：规则中写明「写记忆前必须申请可写权限」。  
3. 不要把记忆根塞进业务仓库只为「能写」——除非你接受数据进 git 的风险。

## 9. 多项目 / 多 Agent

| 场景 | 做法 |
|------|------|
| 多项目 | 每仓 `.agent-memory-project`；语义带 `--project`；依赖 detect high |
| 多 Agent 同时写 | 允许；INDEX 原子写。**v1 仅一份 working**，同时改 current 可能互盖 → 换任务先 handoff |
| 本机换路径 | 整夹拷贝记忆根，改 `AGENT_MEMORY_ROOT` |

## 10. 接入验收清单

### 10.1 五分钟冒烟

```bash
export AGENT_MEMORY_ROOT="/你的/记忆根"
export PATH="/你的/agent_memory/.venv/bin:$PATH"

agent-memory doctor
agent-memory context --query "smoke"
agent-memory checkpoint --goal "接入冒烟"
agent-memory context --query "smoke"    # Working 应含目标
agent-memory handoff --goal "G1 接入" --next-steps "- 下一步"
agent-memory remember --slot smoke_pref --content "接入测试偏好"
agent-memory search "smoke_pref"
```

### 10.2 换 Agent 接续（L-Ref 思想）

1. Agent A：`handoff --goal "G1MARK …" --next-steps "- STEP1MARK …"`  
2. **新会话** Agent B：只跑 `context`，**不要**粘贴 A 的聊天  
3. B 应能从 Working/Handoff 恢复 G1MARK / STEP1MARK  

自动化 L-Core 子集：

```bash
cd /path/to/agent_memory
bash scripts/demo_ac1.sh
```

### 10.3 业务仓检查

- [ ] 存在 `.agent-memory-project`  
- [ ] `AGENTS.md`（或等价）含强制 `context` / checkpoint / 禁编造  
- [ ] 路径与本机一致  
- [ ] `project-detect` 在仓内为 high  
- [ ] 写命令在目标 Agent 上至少成功过一次

## 11. 固定记忆怎么在接入后写入

接入完成后，**全局习惯**用 `remember`（不要 `--project`）；**仅某仓**加 `--project`。

可在任意已接入 Agent 中发指令，或本机终端执行。示例见 [使用手册 §4.7](使用手册.md) 与上文 Codex 话术。

硬约束（每轮必带、宜短）优先改：

```text
$AGENT_MEMORY_ROOT/profile/me.T0.md
```

## 12. 排错（接入向）

| 现象 | 处理 |
|------|------|
| Agent 从不调用 memory | 检查规则是否进常驻上下文；是否被更长 AGENTS 淹没 |
| 只读成功、写入 Permission denied | 沙箱升权 / 可写路径 |
| 写到了错误的根 | 打印 `echo $AGENT_MEMORY_ROOT`；命令加 `--root` |
| 项目语义写不进 | `project-detect`；标记文件；working.project_id |
| 换 Agent 接不上 | 是否执行了 handoff/checkpoint；B 是否只靠 context |
| 与内置 Memory 冲突 | 以本系统文件为准；避免两套各写各的真相 |

更多 CLI 排错见 [使用手册 §8](使用手册.md)。

## 14. 接入完成后的最小日常

1. 新会话 → `context`  
2. 开发中 → 定期 `checkpoint`  
3. 说记住 → `remember`  
4. 换工具 → `handoff`  
5. 偶尔 → `doctor`

详细命令参数见 **[使用手册](使用手册.md)**。
