## 1. 安装

### 1.1 环境要求

- Python **3.10+**
- 可写本机目录（记忆根）
- macOS / Linux 常用；Windows 可用，注意路径与 shell 差异

### 1.2 从源码安装（推荐）

```bash
git clone https://github.com/valiantyan/agent_memory.git
cd agent_memory
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
agent-memory --version
# 期望输出示例：agent-memory 1.0.1 (schema 1.0.0)
```

`pip install -e` 表示可编辑安装：改仓库代码后一般无需重装即可生效。

### 1.3 仅运行依赖

```bash
pip install -e .
# 开发与跑测试再加：
pip install -e ".[dev]"
```

### 1.4 安装 Codex Hooks（推荐，随仓库交付）

装好 CLI 并 `init` 记忆根后，**一条命令**接入 Codex 自动读/写任务态：

```bash
export AGENT_MEMORY_ROOT="${AGENT_MEMORY_ROOT:-$HOME/.agent-memory}"
bash scripts/install_codex_hooks.sh
```

| 会安装什么 | 作用 |
|------------|------|
| `~/.codex/hooks/agent-memory/*.sh` | 开聊 `context`、结束 `checkpoint` |
| 合并 `~/.codex/hooks.json` | **不覆盖**你已有的其它 hooks |
| `*.path` 指针文件 | 记录 CLI 与记忆根路径，GUI 不继承 zshrc 也能找到 |

可选：`bash scripts/install_codex_hooks.sh --with-prompt-search`  
卸载：`bash scripts/uninstall_codex_hooks.sh`  

说明见 `scripts/codex-hooks/README.md`。

## 2. 配置记忆根目录

### 2.1 环境变量

| 变量 | 含义 |
|------|------|
| `AGENT_MEMORY_ROOT` | 记忆数据根目录。未设置时默认 `~/.agent-memory` |

```bash
# 默认位置
export AGENT_MEMORY_ROOT="$HOME/.agent-memory"

# 或自定义（示例）
export AGENT_MEMORY_ROOT="$HOME/notes/grok-memory"
```

建议写入 `~/.zshrc` / `~/.bashrc`，并保证 CLI 在 `PATH` 中：

```bash
export AGENT_MEMORY_ROOT="$HOME/.agent-memory"
export PATH="/绝对路径/agent_memory/.venv/bin:$PATH"
```

### 2.2 `--root` 参数

任意命令都可显式指定根（**写在子命令前或后均可**，≥1.0.1）：

```bash
agent-memory --root "$HOME/.agent-memory" context --query "测试"
agent-memory context --query "测试" --root "$HOME/.agent-memory"
```

未设置环境变量且不传 `--root` 时，使用默认 `~/.agent-memory`。若该处未 `init`，写操作会报 `no schema_version`。

### 2.3 初始化

```bash
agent-memory init
agent-memory doctor    # 应看到 doctor: ok
```

- 已初始化再次 `init` 会拒绝覆盖（保护数据）。  
- 仅在「无记忆正文」等安全条件下可用 `--force` 补模板，**不会清空已有记忆**（详见 `init --help`）。

### 2.4 建议先改 T0

编辑：`$AGENT_MEMORY_ROOT/profile/me.T0.md`

放入**短而硬**的约束，例如：禁止把密钥写入记忆、无检索命中不编造、回复语言偏好等。  
T0 每次 `context` 都会注入（有长度预算，过长会截断）。

## 4. 命令详解

下列路径均相对于记忆根。完整协议义务见 `PROTOCOL.md`。

### 4.1 `init` — 创建空库

```bash
agent-memory init
agent-memory init --force   # 仅安全补模板场景
```

创建 schema、T0 模板、空 INDEX、working 骨架、目录树等。

---

### 4.2 `doctor` — 健康检查

```bash
agent-memory doctor
```

检查 schema、INDEX 与正文一致性、明显异常、高危串等。  
日常在 `reindex` 或手工改文件后建议执行。

---

### 4.3 `reindex` — 重建 INDEX

```bash
agent-memory reindex
```

从正文文件重建 `INDEX.semantic.md` / `INDEX.episodic.md`。  
手工挪文件或怀疑 INDEX 损坏时使用。

---

### 4.4 `context` — 开聊打包（最常用）

```bash
agent-memory context
agent-memory context --query "播放 卡顿 Media3"
agent-memory context --project kmp-music --query "架构"
agent-memory context --top-k 5
agent-memory context --include-staging   # 一般调试用
```

**输出结构（稳定）：**

```text
## T0
...
## Working
...
## Semantic (top_k=N)
### <id> — <one_liner>
<body>
```

| 参数 | 说明 |
|------|------|
| `--query` | 语义检索关键词；空则按更新时间等规则取 top_k |
| `--project` | 覆盖当前项目 scope |
| `--top-k` | 语义条数上限（默认 5） |
| `--include-staging` | 附加候选（默认不包含） |

**用法建议：** 每个新会话开始立刻跑一次；无 Semantic 命中时**不要**对 Agent 说「记忆里有某某」。

---

### 4.5 `search` — 检索

```bash
agent-memory search "Media3"
agent-memory search "偏好" --mode semantic --top-k 10
agent-memory search "上周" --mode episodic
agent-memory search "x" --project kmp-music
agent-memory search "x" --include-staging
agent-memory search "x" --history
```

| 参数 | 说明 |
|------|------|
| `query` | 查询串 |
| `--mode semantic\|episodic` | 语义或情节 |
| `--project` | 项目过滤 / 当前项目 |
| `--top-k` | 条数 |
| `--include-staging` | 含候选 |
| `--history` | 含历史/superseded 等（按实现） |

默认 scope：`global` ∪ 当前项目（high 识别时）。

---

### 4.6 `get` — 按 id 查看

```bash
agent-memory get sem_20260717_xxxx
agent-memory get handoff_20260717150030 --json
```

用于打开 search/recent 得到的 id。

---

### 4.7 `remember` — 立刻写入正式语义（固定记忆）

```bash
# 全局偏好（默认 scope=global）
agent-memory remember \
  --slot coding \
  --content "只改与任务相关的文件；禁止顺手重构与无关格式化"

# 指定类型
agent-memory remember \
  --slot architecture_boundary \
  --content-kind constraint \
  --project kmp-music \
  --content "commonMain 不引入平台 API"

# 可选
agent-memory remember --slot fruit --content "喜欢橘子" --title "水果" --one-liner "当前喜欢橘子"
```

| 参数 | 说明 |
|------|------|
| `--slot` | **必填**。同 scope 下唯一 active；再写则 supersede |
| `--content` | **必填**。正文 |
| `--project` | 项目 id；与 `--scope` 配合写项目语义 |
| `--scope` | 默认 `global` |
| `--content-kind` | 如 preference / constraint / fact / decision… |
| `--title` / `--one-liner` | 标题与一行摘要 |
| `--force` | 仅用于可强制的误报；**密钥类禁止靠 force 绕过** |

**固定记忆配方（示例）：**

```bash
agent-memory remember --slot coding --content "最小改动；根因优先；不覆盖用户未提交改动"
agent-memory remember --slot discussion --content "先结论后依据；默认简洁"
agent-memory remember --slot commit --content "中文说明：问题原因、解决方案、影响范围"
agent-memory remember --slot bugfix --content "复现→根因→最小修复→匹配验证→再声称修好"
```

项目专属：

```bash
agent-memory remember --slot project_identity --project my-app \
  --content-kind fact \
  --content "仓库路径 …；产品定位 …"
```

---

### 4.8 `forget` — 删除记忆

```bash
agent-memory forget sem_xxx          # 软删：退出默认检索
agent-memory forget sem_xxx --hard   # 硬删文件
```

---

### 4.9 `recent` — 最近写入（只读）

```bash
agent-memory recent
agent-memory recent --n 20
```

列出最近操作的 id / 时间 / kind / path。  
**不写盘**；过期日志清理请用 `gc`。

---

### 4.10 `checkpoint` — 更新任务态

```bash
agent-memory checkpoint \
  --goal "修复播放卡顿" \
  --decisions "- 先查 Media3 会话恢复" \
  --next-steps "- 写复现步骤"$'\n'"- 跑相关测试" \
  --project-id kmp-music \
  --session-id "可选会话号"
```

| 参数 | 说明 |
|------|------|
| `--goal` | 当前目标 |
| `--decisions` / `--decisions-file` | 决策 |
| `--next-steps` | 下一步 |
| `--project-id` | 绑定项目 |
| `--session-id` | 可选 |
| `--related-id` | 可多次，关联记忆 id |
| `--force` | 按实现绕过部分检查 |

**建议：** 约每 8 条用户消息、或重大决策/测通后调用。

---

### 4.11 `handoff` — 交接快照

```bash
agent-memory handoff \
  --goal "修复播放卡顿" \
  --next-steps "- STEP1 继续查缓冲"$'\n'"- STEP2 补测试" \
  --project-id kmp-music
```

生成 `working/handoff-*.md`。换 Agent / 换窗口前调用。  
下一会话用 `context` 应能读到 goal / next steps。

---

### 4.12 `session-end` — 写情节摘要

```bash
agent-memory session-end \
  --title "播放卡顿排查" \
  --body "意图：…"$'\n'"动作：…"$'\n'"结果：…"$'\n'"教训：…" \
  --project-id kmp-music
```

| 参数 | 说明 |
|------|------|
| `--title` | 必填 |
| `--body` 或 `--body-file` | 正文（有长度上限） |
| `--one-liner` | 可选摘要 |
| `--project-id` / `--session-id` | 可选 |

**注意：** 写的是**摘要**，禁止把整段聊天当记忆。

---

### 4.13 `extract` — 从情节抽候选

```bash
agent-memory extract --from ep_xxx --mode rules
agent-memory extract --from ep_xxx --mode fixture   # 测试夹具
```

候选进入 `staging/`，**默认不进** search。需人工或门禁 `promote`。

---

### 4.14 `promote` / `reject`

```bash
agent-memory promote cand_id
agent-memory promote cand_id --user-confirmed
agent-memory promote cand_id --related-episode ep1 --related-episode ep2
agent-memory reject cand_id
```

- `promote`：候选 → active（程序记忆有额外确认要求）。  
- `reject`：登记后不再自动转正。

---

### 4.15 `gc` — 垃圾回收

```bash
agent-memory gc
agent-memory gc --dry-run
```

惰性到期处理、情节归档、**裁剪** `recent` 日志等。

---

### 4.16 `project-detect` — 项目识别

```bash
agent-memory project-detect
agent-memory project-detect /path/to/repo
agent-memory project-detect --effective
agent-memory project-detect --force-confidence=low   # 仅测试/调试
```

输出 `project_id` 与 `confidence`（`high` / `low`）。  
**low 时禁止写入 project 范围正式语义**（仍可写 episode / working / global）。

项目标记文件（在业务仓库根）：

```text
.agent-memory-project
```

内容一行项目 id，例如：`kmp-music`。

## 5. 日常使用配方

### 5.1 新开一天 / 新会话

```bash
agent-memory context --query "今天要做的事的关键词"
# 阅读 T0、Working、Semantic 后再动手
```

### 5.2 用户说「记住…」

```bash
agent-memory remember --slot <稳定英文或拼音槽位> --content "……"
```

### 5.3 开发中途

```bash
agent-memory checkpoint --project-id <id> --goal "…" --next-steps "…"
```

### 5.4 换工具 / 换窗口

```bash
agent-memory handoff --goal "…" --next-steps "- …" --project-id <id>
# 新会话：
agent-memory context --query "…"
```

### 5.5 收工

```bash
agent-memory session-end --title "…" --body "意图/动作/结果/教训"
# 可选：
agent-memory extract --from <episode_id> --mode rules
# 审查 staging 后 promote 或 reject
```

### 5.6 忘掉

```bash
agent-memory recent --n 10
agent-memory forget <id>
```

### 5.7 整库体检

```bash
agent-memory doctor
agent-memory reindex   # 仅当怀疑 INDEX 不一致
```

## 6. 范围与隔离

| 写入 | 命令要点 | 落点 |
|------|----------|------|
| 全局习惯 | `remember` 不带 `--project` | `scopes/global/semantic/` |
| 项目约定 | `remember --project <id>` 且 detect high | `scopes/projects/<id>/semantic/` |
| 当前任务 | `checkpoint` | `working/current.md` |
| 交接 | `handoff` | `working/handoff-*.md` |
| 情节 | `session-end` | `episodes/…` |

**不要**把密钥、Token、Cookie、私钥写入任何记忆文件。  
**不要**把完整聊天记录当 semantic。

## 8. 排错

| 现象 | 处理 |
|------|------|
| `no schema_version; run init` | 对正确根执行 `init`，或检查 `AGENT_MEMORY_ROOT` / `--root` |
| `command not found: agent-memory` | `source` venv 或把 `.venv/bin` 加入 PATH；或用绝对路径 |
| `unrecognized arguments: --root` | 升级到 ≥1.0.1；或把 `--root` 放在子命令前 |
| `Operation not permitted`（沙箱） | 记忆根在项目外时，写命令需升权；`context`/`search`/`recent`/`doctor` 只读通常可用 |
| project 写入被拒 | `project-detect` 是否 high；是否有 `.agent-memory-project`；`checkpoint --project-id` |
| 密钥被拦截 | 从 content 去掉密钥；不要用 `--force` 硬塞密钥 |
| search 不到刚写的 | `doctor`；是否 soft-forget；scope 是否 project 过滤错了 |
| INDEX 可疑 | `reindex` 后 `doctor` |

## 10. 字符与预算（摘要）

系统用 Python `len(str)` 计「字符」。常用默认：

| 项 | 约值 |
|----|------|
| T0 段 | ≤ 1600 |
| context 语义详情合计 | ≤ 4800 |
| 语义 top_k | 5 |
| recent 默认条数 | 20 |
| recent 保留 | ≥ 30 自然日（由 gc 裁剪文件） |

细节见 `REQUIREMENTS.md` §9。

## 11. 安全清单

1. 记忆根权限仅本人可读更佳（init 后注意文件模式）。  
2. 勿提交 `AGENT_MEMORY_ROOT` 到公开仓库。  
3. 定期 `recent` / `search` 自查是否误记隐私。  
4. 误记后立刻 `forget --hard` 并检查备份。
