# 四个问题的具体解决路径（不改 / 少改核心 CLI）

| 项 | 内容 |
|----|------|
| 适用 | agent-memory **1.0.1** + Codex（含 hooks） |
| 原则 | 人与 Agent **产数据**；记忆系统 **存/甄/滤/洗**；Hooks 只做 **时机触发** |
| 核心代码 | **默认不改** `agent_memory/`；靠协议 + hooks + 已有命令 |

---

## 0. 四个问题 → 解法映射

| # | 你的问题 | 合格标准 | 具体靠什么解决 |
|---|----------|----------|----------------|
| **Q1** | 每轮结束后自动写精华，且不当垃圾桶；换会话知道未完成任务 | 每轮刷新 **Working**；长期语义严进 | **Stop hook** + 协议强制 `checkpoint`；`remember` 仅明确长期 |
| **Q2** | 人定位=和 Agent 产数据；记忆=保存+甄别+过滤+清洗 | 分层写入、有门禁 | **用现有分层**（见 §2）+ 写入规则写死在协议/hook |
| **Q3** | 用户发话→Agent 说完可用 hooks 监听 | 有稳定触发点 | **Codex `UserPromptSubmit` / `Stop` / `SessionStart`** |
| **Q4** | 发消息后检索历史，帮回忆以前怎么做 | 开聊必读；相关再搜 | **SessionStart → context**；条件 **UserPrompt → search/context** |

```text
SessionStart ──► context（读：T0 + Working + 相关语义）     ← Q4
UserPrompt  ──► 条件检索（可选）                            ← Q4
   … Agent 工作，人与 Agent 产生数据 …                     ← Q2
Stop        ──► checkpoint（写：任务态精华）± 严控 remember  ← Q1、Q2、Q3
```

## 1. 总体架构（落地版）

```text
┌──────────────┐     hooks      ┌─────────────────────┐     CLI      ┌──────────────────┐
│ Codex 会话    │ ────────────► │ 适配脚本（本机）        │ ─────────► │ agent-memory     │
│ 人 ↔ Agent   │               │ session_start / stop  │            │ 分层存储与门禁    │
└──────────────┘               └─────────────────────┘            └──────────────────┘
```

| 层 | 做什么 | 不做什么 |
|----|--------|----------|
| Codex hooks | 在对的时间调脚本 | 不当第二记忆库 |
| 适配脚本 | 调 CLI、限长、选命令 | 不把 JSONL 全文入库 |
| agent-memory | 存 Working / 语义 / 清洗 | 不解析聊天流（v1） |
| AGENTS.md | 规定 Agent 必须写结构化精华、何时 remember | — |

## 2. Q2：甄别规则（先写死，再自动化）

**每一回合结束，数据只允许进这些「桶」：**

| 桶 | 命令 | 自动？ | 内容 |
|----|------|--------|------|
| **任务态** | `checkpoint` | **是（每轮）** | goal、next_steps、本轮 decisions（短） |
| **交接** | `handoff` | 换工具/收工 | 给下一 Agent 的快照 |
| **情节摘要** | `session-end` | 可选，一天/大任务一次 | 意图/动作/结果/教训（摘要） |
| **长期约定** | `remember` | **否**（除非用户说记住，或你明确规则） | 稳定偏好/项目约定 |
| **丢弃** | 不写 | — | 闲聊、试错细节、密钥、全文对话 |

这就是「像人一样该记记、该扔扔」——**规则在适配层+协议，执行在 CLI。**

## 3. 分步实施（按顺序做）

### 步骤 A — 环境一次配好（5 分钟）

```bash
# 1) CLI 可用
export AGENT_MEMORY_ROOT="/Users/yanhao/Downloads/github-nots/notes/grok-memory"  # 按你的实际根改
export PATH="/Users/yanhao/Downloads/grok/.venv/bin:$PATH"
agent-memory --version
agent-memory doctor

# 2) 项目有标记（示例 kmp-music）
# 仓库根 .agent-memory-project 内容一行：kmp-music
```

写入 `~/.zshrc`（若尚未写）。

### 步骤 B — 业务仓协议（解决「Agent 有语义理解时写什么」）

在项目 `AGENTS.md` **外挂记忆**一节追加（或等价规则）：

```markdown
### 回合读写（强制）

**读（会话开始 / 恢复）：**
1. 先执行：`agent-memory context --query "<用户首条或当前任务关键词>"`
2. 无 Semantic 命中时禁止编造历史做法。

**写（每个 user 回合结束前必须做完）：**
1. 执行 checkpoint，只写任务精华（不要把对话全文写进 content）：
   `agent-memory checkpoint --project-id <id> --goal "…" --next-steps "- …" --decisions "- …"`
2. 仅当用户明确「记住/以后都/别再」时：
   `agent-memory remember --slot <slot> --content "…"`
3. 换窗口/换 Agent/收工：`handoff`；大段落结束可 `session-end`（摘要）。

**禁止：** 每轮 remember；密钥入库；无 hit 假装有记忆。
```

→ 这解决：**Q1 的语义质量**（模型懂本轮做了什么）+ **Q2 甄别** + **Q4 开聊读**。

### 步骤 C — Codex Hooks（解决「自动触发」Q3）· **正式交付一键安装**

仓库已写好完整 hooks + 安装器（**使用者装记忆系统时直接装**，不必手改 JSON）：

```bash
export AGENT_MEMORY_ROOT="…"   # 你的记忆根
bash scripts/install_codex_hooks.sh
# 可选条件检索：
# bash scripts/install_codex_hooks.sh --with-prompt-search
```

| 事件 | 脚本 | 作用 |
|------|------|------|
| `SessionStart` | `session_start.sh` | `context` → **stdout 注入** + 私有缓存 |
| `Stop` | `stop_turn.sh` | **数据根** `meta/pending-turn/`（`agent-memory turn`）→ `checkpoint`；否则 **no-op** |
| `UserPromptSubmit`（可选） | `user_prompt_maybe_search.sh` | 关键词才再 context 并注入 |

安装器会：**复制脚本、写入 CLI/记忆根路径、命令级合并 hooks.json、备份轮转、保留 Muxy 等已有 hook**。  
卸载：`bash scripts/uninstall_codex_hooks.sh [--purge-cache]`。  
说明：`scripts/codex-hooks/README.md`。

### 步骤 D — 回合精华契约（v2 · 只写数据根）

Agent **结束前**（不依赖业务仓目录）：

```bash
agent-memory turn --goal "修复播放卡顿" --next-steps $'- 补复现\n- 跑测试' --cwd .
```

写入 **`$AGENT_MEMORY_ROOT/meta/pending-turn/<project>.json`**。

- Stop：claim → checkpoint → `done/`  
- 无 pending / 不完整 → **no-op**（不冲 Working）

### 步骤 E — 换电脑 / 换人

```bash
# 旧机器：备份整个记忆根
cp -a "$AGENT_MEMORY_ROOT" ~/backup-agent-memory

# 新机器：装 CLI、设 AGENT_MEMORY_ROOT 指向拷贝目录
agent-memory doctor
agent-memory context --query "当前项目"
```

未完成任务在 **Working**；不靠 Codex 会话文件搬家。

### 步骤 F — 验收（30 分钟内可测）

| 测项 | 操作 | 期望 |
|------|------|------|
| 开聊能回忆任务 | 新会话只靠 context | 看到上次 goal/next |
| 一轮开发后 | 看 `working/current.md` 时间戳与内容 | 已更新 |
| 不进垃圾桶 | `search` 不应出现整段闲聊 | 无全文对话 |
| 说记住 | remember 后 context/search 能命中 | 长期约定在 |
| 换会话 | 不粘贴聊天，只 context | 能接 next_steps |
| hooks | 关/开 Codex 跑一轮 | Stop 后 recent 有 checkpoint |

```bash
agent-memory recent --n 10
agent-memory context --query "当前"
```

## 4. Q4 检索：具体何时查

| 时机 | 动作 | 强度 |
|------|------|------|
| SessionStart / 首条 | `context --query "…"` | **强制** |
| 用户含「上次/之前/怎么做/坑」 | 再 `context` 或 `search` | **强制** |
| 普通「继续改这个函数」 | 依赖已注入 Working + 代码 | 不强制全库搜 |
| 每条消息无脑 full context | — | **不做**（省 token） |

条件检索可在 `UserPromptSubmit` hook 里用简单关键词；复杂语义仍交给 Agent 内再 `search`。

## 6. 和「改记忆系统代码」的边界

| 以后若仍痛 | 再考虑核心改动 |
|------------|----------------|
| checkpoint 参数太长 Agent 常拼错 | 增加 `checkpoint --from-json file` |
| 想官方一键安装 hooks | 增加 `scripts/install_codex_hooks.sh` |
| 要 CLI 内解析 Codex 会话 | 大功能，单独立项 |

**按本文做完，四个问题在工程上即可闭环，无需先改核心。**

## 8. 一句话

| 问题 | 一句话解法 |
|------|------------|
| Q1 自动精华不进垃圾桶 | 每轮只自动 `checkpoint` Working；长期靠 remember 门禁 |
| Q2 产 vs 存甄滤洗 | 人+Agent 产；CLI 分层+门禁；规则写在协议/hook |
| Q3 hooks | SessionStart 读、Stop 写；接现有 Codex hooks |
| Q4 检索回忆 | 开聊必 context；相关再 search；禁止每句全量无脑搜 |
